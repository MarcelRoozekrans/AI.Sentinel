# MCP Proxy v1.1 Design

**Date:** 2026-04-26

---

## Problem

The v1 MCP proxy (`AI.Sentinel.Mcp`) intercepts `tools/call` and `prompts/get` over stdio only. Ten backlog items have accumulated since v1 shipped:

- A real coverage gap: `resources/read` is a third content-bearing path through which a malicious or compromised MCP server can deliver injection payloads, and the proxy currently has zero visibility into it.
- A capability mismatch: the proxy unconditionally advertises `Tools` + `Prompts` to the host, regardless of what the target supports. Wiring the proxy in front of a prompts-only server produces an empty `tools/list` or runtime error.
- A transport limitation: stdio-only means the proxy can't sit in front of hosted MCP servers (Cloudflare-style endpoints, internal HTTP services).
- Robustness gaps: subprocess hangs leak the child process; the `CancelKeyPress` handler is never unsubscribed (leaks per call when hosted in long-lived processes); env-var-only severity config doesn't match conventional CLI ergonomics; the size cap counts chars instead of bytes (silently misleading on non-ASCII content).
- Observability gaps: per-request stderr lines are `key=value` rather than NDJSON (annoys log aggregators); the AOT smoke test only exercises `--help`.
- Test gaps: all proxy tests are sequential; no concurrent / adversarial coverage.

## Goal

Ship a coherent v1.1 release that closes the `resources/read` gap, fixes capability mirroring, adds StreamableHttp transport, hardens subprocess shutdown, and clears the polish backlog. All changes are additive (no breaking API changes); existing stdio + tool-call + prompts-get behaviour is preserved.

**Three structural changes:**

1. **`ResourceReadInterceptor`** â€” new file. Per-content-item: MIME allowlist check â†’ byte-count cap check â†’ scan as `ChatRole.Tool` message via `SentinelPipeline.ScanMessagesAsync`. Skip + stderr log on miss.
2. **Transport dispatch in `McpProxy.RunAsync`** â€” sniff target string: `http://` / `https://` â†’ `StreamableHttpClientTransport` (with `SENTINEL_MCP_HTTP_HEADERS` parsed); else â†’ existing `StdioClientTransport`.
3. **Capability mirroring** â€” after `McpClient.CreateAsync`, read `targetClient.ServerCapabilities`. Advertise `Tools` / `Prompts` / `Resources` to the host only if (a) target supports it AND (b) we have an interceptor for it.

Plus a small extracted `StderrLogger` that supports both `key=value` and NDJSON output, replacing scattered `Console.Error.WriteLine(...)` calls.

## Per-Item Specification

### 1. `resources/read` interception

**File:** new `src/AI.Sentinel.Mcp/ResourceReadInterceptor.cs` â€” mirrors `ToolCallInterceptor`'s `Create(...)` factory shape.

**Algorithm (per `ResourceContent` item in the response):**
1. Resolve the active MIME allowlist (default: `text/*`, `application/json`, `application/xml`, `application/yaml`; override via `SENTINEL_MCP_SCAN_MIMES` env var, comma-separated, `text/*` wildcard supported).
2. If MIME doesn't match â†’ forward unscanned, stderr log: `mime=image/png event=resources_read action=skipped reason=mime`.
3. If `Encoding.UTF8.GetByteCount(text) > SENTINEL_MCP_MAX_SCAN_BYTES` â†’ forward unscanned, stderr log: `event=resources_read action=skipped reason=oversize bytes=...`.
4. Scan via `SentinelPipeline.ScanMessagesAsync` with `[new ChatMessage(ChatRole.Tool, text)]`.
5. On `SentinelError.ThreatDetected` â†’ throw `McpProtocolException(InternalError, ...)` (matches existing block path on `tools/call`).
6. On any other exception â†’ fail open, log to stderr (matches existing fail-open posture).

### 2. Capability mirroring

**File:** modify `src/AI.Sentinel.Mcp/McpProxy.cs`.

After `McpClient.CreateAsync(targetTransport, ...)`, read `targetClient.ServerCapabilities`:

```csharp
var serverOptions = new McpServerOptions { ServerInfo = ... };
if (targetClient.ServerCapabilities.Tools is not null)
    serverOptions.Capabilities.Tools = new() { /* + ToolCallInterceptor */ };
if (targetClient.ServerCapabilities.Prompts is not null)
    serverOptions.Capabilities.Prompts = new() { /* + PromptGetInterceptor */ };
if (targetClient.ServerCapabilities.Resources is not null)
    serverOptions.Capabilities.Resources = new() { /* + ResourceReadInterceptor */ };
```

Resources advertised iff (target has them) AND (we wire the interceptor) â€” closes the loop with item 1.

### 3. StreamableHttp transport

**File:** modify `src/AI.Sentinel.Mcp/McpProxy.cs`.

```csharp
IClientTransport transport = targetCmd.StartsWith("http://", Ordinal)
                          || targetCmd.StartsWith("https://", Ordinal)
    ? new StreamableHttpClientTransport(new() {
          Endpoint = new Uri(targetCmd),
          AdditionalHeaders = ParseHeaders(Environment.GetEnvironmentVariable("SENTINEL_MCP_HTTP_HEADERS")),
      })
    : new StdioClientTransport(new() { Command = targetCmd, Arguments = restArgs });
```

`ParseHeaders("Authorization=Bearer xyz;X-Tenant=acme")` â†’ `Dictionary<string, string>` keyed Ordinal. Empty / null env var â†’ empty dict.

**Out of scope (deferred):** `SseClientTransport`, OAuth2 flow, mTLS.

### 4. Subprocess lifecycle hardening

**File:** modify `src/AI.Sentinel.Mcp/McpProxy.cs`.

Wrap `StdioClientTransport` disposal:

```csharp
async Task DisposeWithGraceAsync(IClientTransport t, TimeSpan grace)
{
    var disposeTask = t is IAsyncDisposable ad ? ad.DisposeAsync().AsTask() : Task.Run(() => t.Dispose());
    if (await Task.WhenAny(disposeTask, Task.Delay(grace)) != disposeTask)
    {
        // Grace expired â€” kill the child process. StdioClientTransport doesn't expose the Process
        // directly; we track it by capturing the process spawn elsewhere or by using a custom transport
        // wrapper. Simplest: subclass StdioClientTransport or track via Process.GetProcessById.
        StderrLogger.Log("event=subprocess_kill grace_sec={grace}");
        // ... kill logic ...
    }
}
```

Default grace: **5 seconds**. Override: `SENTINEL_MCP_TIMEOUT_SEC=N`. Applies to shutdown only â€” not per-request timeouts (those flow through the SDK's `CancellationToken` and are out of scope).

### 5. CLI severity flags

**File:** modify `src/AI.Sentinel.Mcp.Cli/Program.cs`.

Add four flags: `--on-critical`, `--on-high`, `--on-medium`, `--on-low`, each accepting `Block | Warn | PassThrough`. **Precedence:** CLI flag > env var > default.

```bash
sentinel-mcp --on-critical Block --on-high Warn -- /path/to/server arg1 arg2
sentinel-mcp --on-critical Block --target https://example.com/mcp
```

### 6. Structured JSON logging

**File:** new `src/AI.Sentinel.Mcp/Logging/StderrLogger.cs`. Replaces scattered inline `Console.Error.WriteLine(...)` calls.

```csharp
internal static class StderrLogger
{
    private static readonly bool UseJson =
        string.Equals(Environment.GetEnvironmentVariable("SENTINEL_MCP_LOG_JSON"), "1", StringComparison.Ordinal);

    public static void Log(IReadOnlyDictionary<string, string> fields) =>
        Console.Error.WriteLine(UseJson ? Serialize(fields) : Format(fields));
}
```

Same fields, two output shapes. NDJSON enables direct ingestion by log aggregators (Loki, ELK).

### 7. Real-bytes truncation

**File:** modify wherever `SENTINEL_MCP_MAX_SCAN_BYTES` is consumed (likely `ToolCallInterceptor.cs` and the new `ResourceReadInterceptor.cs`).

Replace `text.Length > maxBytes` with `Encoding.UTF8.GetByteCount(text) > maxBytes`. Behaviour change: cap fires earlier on multi-byte content. ASCII unchanged. Documented in README + commit message â€” no env-var rename (same name, more accurate semantics).

### 8. `CancelKeyPress` handler scoping

**Files:** `src/AI.Sentinel.Mcp.Cli/Program.cs`, `src/AI.Sentinel.ClaudeCode.Cli/Program.cs`, `src/AI.Sentinel.Copilot.Cli/Program.cs`.

```csharp
ConsoleCancelEventHandler handler = (_, _) => cts.Cancel();
Console.CancelKeyPress += handler;
try { /* RunAsync */ }
finally { Console.CancelKeyPress -= handler; }
```

Harmless in one-shot CLI; fixes leak when the CLI is hosted inside a longer-lived process (integration tests).

### 9. Concurrency + adversarial test coverage

**File:** new tests in `tests/AI.Sentinel.Tests/Mcp/`.

Add ~5-6 tests:
- `ConcurrentToolCalls_AllScanned` â€” `Task.WhenAll` 10Ă— `CallToolAsync` against the same proxy + target; assert all complete + all scanned (no missed audit entries).
- `Adversarial_TargetReturnsErrorWithInjectionText` â€” target returns `IsError=true, Content=[ "ignore all previous instructions and ..." ]`; assert detection still fires on the error content.
- `Adversarial_TargetReturnsUnparseableBlob` â€” target returns malformed content; assert proxy doesn't crash, fails open with stderr log.
- `AllPreset_RoundTripScansThroughAllDetectors` â€” wire the proxy with `McpDetectorPreset.All`, run a clean `tools/call` round-trip, assert no false positives.
- `AllPreset_ExpensiveSemanticDetectors_DontTimeOut` â€” same as above with a `FakeEmbeddingGenerator` to exercise the semantic path.

### 10. AOT nested-JSON smoke test

**File:** modify `.github/workflows/aot.yml` (or wherever the AOT smoke test lives).

Today the workflow only verifies the published binary's `--help` path. Extend to do one nested-JSON `CallToolAsync` round-trip against the AOT-published binary. Confirms `ToObjectDictionary`'s boxing doesn't trip AOT-incompatible reflection inside the SDK.

## Configuration Summary

### New env vars (additive)

| Var | Default | Purpose |
|---|---|---|
| `SENTINEL_MCP_SCAN_MIMES` | `text/*,application/json,application/xml,application/yaml` | Comma-separated MIME allowlist for `resources/read`. `text/*` wildcard supported. |
| `SENTINEL_MCP_HTTP_HEADERS` | (none) | `key=value;key=value;...` headers applied to StreamableHttp requests. |
| `SENTINEL_MCP_TIMEOUT_SEC` | `5` | Subprocess shutdown grace before SIGKILL. |
| `SENTINEL_MCP_LOG_JSON` | (off) | `1` = NDJSON stderr; else `key=value`. |

### New CLI flags (additive)

```
sentinel-mcp [--on-critical Block|Warn|PassThrough]
             [--on-high     Block|Warn|PassThrough]
             [--on-medium   Block|Warn|PassThrough]
             [--on-low      Block|Warn|PassThrough]
             -- <stdio-target-cmdâ€¦>                            # stdio mode
sentinel-mcp [...flags...] --target https://example.com/mcp   # http mode
```

Precedence: CLI flag > env var > default. (Today's env vars `SENTINEL_MCP_ON_CRITICAL` etc. continue to work.)

### Behaviour change (one)

`SENTINEL_MCP_MAX_SCAN_BYTES` now counts UTF-8 bytes (was char count). For ASCII content: identical. For multi-byte: cap fires sooner. Documented in README; same env var name preserved.

## Test Strategy

~12-15 new tests in `tests/AI.Sentinel.Tests/Mcp/`:

| Area | Tests |
|---|---|
| `resources/read` | text scanned, JSON scanned, oversized text skipped+logged, image/binary MIME skipped+logged, custom `SCAN_MIMES` honored |
| Capability mirroring | prompts-only target â†’ proxy advertises Prompts only; full target â†’ advertises everything we intercept |
| HTTP transport | URL sniff picks StreamableHttp; `SENTINEL_MCP_HTTP_HEADERS` parsed and applied (config-wiring level; no live HTTP server in unit tests) |
| Subprocess hardening | hung target killed after grace period; `SENTINEL_MCP_TIMEOUT_SEC` honored |
| Polish | NDJSON logging output shape; UTF-8 byte-count truncation cap; CLI severity flag overrides env var |
| Concurrency + adversarial | 10Ă— concurrent `CallToolAsync`; injection text in error message detected; unparseable blob handled |

Plus the AOT workflow extension (CI-side, not a unit test).

## Out of Scope (explicit)

These remain in the backlog after v1.1:

- **`SseClientTransport`** â€” older SSE transport. Deprecated by spec; defer until a real user asks.
- **Per-request timeouts** â€” different concern from shutdown grace; SDK already supports `CancellationToken`.
- **OAuth2 / mTLS auth for HTTP transport** â€” `SENTINEL_MCP_HTTP_HEADERS` covers static-token auth; OAuth flow is its own design.
- **Hot-reload of MIME allowlist / CLI severity** â€” env vars + flags read once at startup. Restart to change.
